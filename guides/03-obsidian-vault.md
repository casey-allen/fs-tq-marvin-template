# Setting Up Your Obsidian Second Brain

> **Reading this on GitHub?** Follow the steps below at your own pace.
> **Inside MARVIN?** Say `/guide obsidian` and I'll walk you through it.

## Prerequisites

- MARVIN set up and running (complete [Guide 01](01-getting-started.md) first)
- [Obsidian](https://obsidian.md) installed (free download)

## What You'll Set Up

By the end of this guide you'll have:
- A git-tracked Obsidian vault with a clean folder structure
- A daily notes template with MARVIN-compatible task tags
- MARVIN configured to read and write to your vault
- A system for capturing tasks, delegations, blockers, and notes that MARVIN can parse

## Why a Second Brain?

Your daily notes capture context that would otherwise be lost between sessions. When you write down what you're working on, what's blocked, and what you've delegated, MARVIN can:
- Pull it into your morning briefing
- Aggregate it into weekly reviews
- Track delegations and follow-ups across people
- Surface stale items you forgot about

The tags you'll learn below (ACTION, DONE, BLOCKED, etc.) are how MARVIN understands your notes.

## Step 1: Create Your Vault

```bash
mkdir -p ~/git/obsidian
cd ~/git/obsidian
git init
```

This creates an empty vault directory tracked by git. We put it in `~/git/` alongside your other repos, but you can put it anywhere.

## Step 2: Create the Folder Structure

```bash
mkdir -p Daily Weekly Templates Fleeting Permanent Meetings People
```

| Folder | Purpose |
|--------|---------|
| `Daily/` | One note per day (YYYY-MM-DD.md) — your primary workspace |
| `Weekly/` | Weekly review summaries generated by MARVIN |
| `Templates/` | Reusable templates (daily note, meeting note, etc.) |
| `Fleeting/` | Quick captures — thoughts, ideas, links to process later |
| `Permanent/` | Reference notes — things you want to keep long-term |
| `Meetings/` | Meeting notes |
| `People/` | Per-person tracking — delegations, 1:1 notes, open items |

## Step 3: Create MARVIN State Files

MARVIN tracks your priorities and goals in your vault's `Permanent/` folder (instead of local files). This means they're accessible from both Obsidian and MARVIN.

Create `Permanent/priorities.md`:
```markdown
# Priorities

Last updated: YYYY-MM-DD

## Active Priorities

1. Complete MARVIN setup
2. [Add your priorities here]

## Open Threads

- None yet

## Recent Context

- Just set up MARVIN!
```

Create `Permanent/goals.md`:
```markdown
# Goals

Last updated: YYYY-MM-DD

## Work Goals

| Goal | Target | Status | Notes |
|------|--------|--------|-------|
| | | | |

## Personal Goals

| Goal | Target | Status | Notes |
|------|--------|--------|-------|
| | | | |
```

MARVIN updates these automatically via `/update` and `/end`. You can also edit them directly in Obsidian anytime.

## Step 4: Create the Daily Note Template

Create the file `Templates/Daily Template.md`:

```markdown
# {{date:YYYY-MM-DD}}

## Today's Focus
-

## Notes

## End of Day
```

This is the template Obsidian will use when you create a new daily note. MARVIN appends to the "Today's Focus" section throughout the day.

## Step 5: Learn the Tag System

These tags are how MARVIN parses your daily notes. Use them inline in your notes:

| Tag | When to Use | Example |
|-----|------------|---------|
| `ACTION:` | A task or reminder | `- ACTION: Review the BGP cutsheet by Friday` |
| `DONE:` | Something you completed | `- DONE: Shipped the OSPF automation script` |
| `BLOCKED:` | Waiting on someone/something | `- BLOCKED: Need carrier to light cross-connect (Zayo, 3 days)` |
| `WAITING:` | Vendor/procurement dependency | `- WAITING: Juniper MX304 linecards (210 days, ordered Nov 15)` |
| `DELEGATE:` | Assigned to someone | `- DELEGATE: @Chris document the fiber shuffle process` |
| `LONG TERM:` | Needs planning, not urgent | `- LONG TERM: Evaluate SD-WAN vendors for corporate network` |

Tips:
- Include context: `BLOCKED: @Platform team to provision the staging VPC — blocks NET-142`
- Include dates: `ACTION: Submit change request by 2026-04-01`
- Include lead times: `WAITING: Fiber splice (vendor: Zayo, ETA: 2 weeks)`

## Step 6: Configure Obsidian

Open Obsidian and point it at your vault:
1. Open Obsidian
2. Click "Open folder as vault"
3. Select `~/git/obsidian`

Then set up the Daily Notes plugin:
1. Go to Settings (gear icon) > Core plugins > Enable "Daily notes"
2. In Daily notes settings:
   - **Date format:** `YYYY-MM-DD`
   - **New file location:** `Daily`
   - **Template file location:** `Templates/Daily Template`

Now when you click the "Open today's daily note" icon (calendar), it creates a note from your template.

## Step 7: Set Up Git Tracking

Back in the terminal:

```bash
cd ~/git/obsidian

# Ignore Obsidian's workspace files (they change constantly and are machine-specific)
cat > .gitignore << 'EOF'
.obsidian/workspace.json
.obsidian/workspace-mobile.json
.trash/
EOF

git add -A
git commit -m "Initial vault setup"
```

**Optional — push to GitHub for backup:**
1. Create a **private** repository on GitHub (e.g., `my-obsidian-vault`)
2. Then:
```bash
git remote add origin git@github.com:YOUR_USERNAME/my-obsidian-vault.git
git push -u origin main
```

## Step 8: Connect to MARVIN

Update your `config.yaml` in your MARVIN workspace to point to the vault:

```yaml
# --- Obsidian ---
obsidian_vault: /Users/YOUR_USERNAME/git/obsidian
daily_notes_dir: Daily/
weekly_notes_dir: Weekly/
templates_dir: Templates/
people_dir: People/
```

Replace `YOUR_USERNAME` with your actual username, or use the full path to wherever you created the vault.

## Verification

Inside MARVIN, run `/start`. Your briefing should now include items from your Obsidian daily note (if you have one for today). If you don't have a note yet, create one in Obsidian, add a line like `- ACTION: Test MARVIN integration`, and run `/start` again.

## Troubleshooting

**MARVIN can't find my vault** — Check that the path in `config.yaml` is absolute (starts with `/`) and points to the correct directory. Run `ls ~/git/obsidian/Daily/` to verify the structure exists.

**Daily notes aren't using the template** — In Obsidian Settings > Daily notes, make sure the template path is `Templates/Daily Template` (no `.md` extension).

**Git complains about large files** — If you store images in your vault, consider adding `*.png` and `*.jpg` to `.gitignore`, or use git-lfs.
